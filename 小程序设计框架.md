## 一、整体架构

### 开屏体验

```
用户首次打开小程序
    ↓
全屏开屏页（5张大图轮播，每2s自动切换，由大变小的zoomIn动画）
    ├── 每张图右下角显示中英文景点名
    ├── 右上角「跳过 Ns」按钮 → 倒计时5秒后可点击/自动进入首页
    └── 底部圆点指示器（当前页宽4px，其余宽1.5px）
```

> **设计意图：** 先用视觉冲击留住用户兴趣，再让用户进入内容。仅首次打开时显示。

### Tab 1：旅游（首页）— 官方推荐驱动的主入口

> **设计理念：** 用户看到的第一眼是"哇这个地方好美"，先留住兴趣，再引导探索。
> 首页信息来源为官方精选/权威评价，确保可靠性和调性统一。

#### 页面模块（从上到下）

| 顺序 | 模块 | 内容 | 状态 |
|:----:|------|------|:----:|
| ① | 顶部导航 | 品牌Logo（R方形图标）+ RotoruaNZ 标题 + 搜索框 + 中/EN切换 | ✅ |
| ② | 快速导航区 | **2×4 网格**，每格图标 + 中英文标签。8个入口：景点 / 住宿 / 美食 / 活动 / 航班 / 路线推荐 / 项目招商 / 租车 | ✅ |
| ③ | Banner | 全宽大图，底部叠加中英双语 + 描述 + 圆点指示器 | 🟡 仅显示第一张（非轮播） |
| ④ | 热门景点 | 横向滚动卡片，带分类标签/评分/价格 | ✅ |
| ⑤ | 玩点什么 | 3个分类网格（地热温泉/毛利文化/户外探险）× 每类4张 | ✅ |
| ⑥ | 罗托鲁瓦小贴士 | 深色底卡片 + 4个标签 | ✅ |
| ⑦ | 精选住宿 | 横向滚动住宿卡片 | ✅ |
| ⑧ | 探索互动地图CTA | 底栏跳转地图页按钮 | ✅ |
| ⑨ | 底部留白 | 安全区域 + 瀑布流 | ✅ |

#### 卡片统一格式

```
全宽图片（h-36，rounded-lg 圆角）
├── 渐变遮罩（from-black/60 从底到顶）
├── 右上角：心形收藏按钮（半透明白底，点击切换 fill 状态）
├── 底部文字浮层
│   ├── 英文名 + ⭐评分（amber色）
│   └── 中文名（左） + 价格（右）
└── fade-in 渐入动画（0.4s，每张延迟60ms）
```

| 数据字段 | 类型 | 示例 |
|---------|------|------|
| id | string | a1, b2 |
| name | string（中文） | 怀奥塔普地热世界 |
| en | string（英文） | Wai-O-Tapu |
| img | string（路径） | /images/waio-1.jpg |
| price | string（价格） | NZ$45 |
| r | number（评分） | 4.9 |

### 全站中英文切换机制

- `t(key)` — 基础字符串翻译
- `td(zh, en)` — 直接双语切换，根据当前 `lang` 返回对应内容
- `tc({ zh, en })` — 对象式翻译
- 右上角按钮 EN/中 一键切换全局语言
- 所有页面、数据、表单、提示均已双语覆盖

## 四、核心转化链路

### 链路一：官方推荐 → 预订（主要转化路径）

```
开屏页（吸引注意）
  ↓ 跳过 / 倒计时结束
首页 → 浏览官方精选推荐（热门景点/玩点什么/精选住宿）
  ↓ 点击卡片
产品详情页 → 浏览图文中英双语
  ↓
预订下单页 → 选日期/人数/套餐
  ↓
订单成功页
  ↓
回到首页 / 前往「我的」查看订单
```

### 链路二：地图发现 → 预订（探索型）

```
地图页 → 缩放浏览/分类筛选
  ↓ 点击标记点
弹出浮卡 → 查看详情/导航
  ↓ 点击详情
产品详情页 → 预订
```

### 链路三：文化发现 → 购买（文化驱动型）

```
特色页 → 浏览商品/文化故事
  ↓ 点击商品
文化商品详情页 → 预订/收藏
  ↓
回到特色页 / 收藏列表
```
